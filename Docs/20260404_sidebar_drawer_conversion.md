# 작업 로그: 2026-04-04 파일 탐색기 독립 및 상단 메뉴 드로어(Drawer)화 작업

## 개요
- **배경**: 에디터와 미리보기의 창 분할(Split) 기능은 코딩 시 매우 유용하므로 유지하되, 쓸모없이 공간을 잡먹고 있던 좌측의 "파일 탐색기(사이드바)"를 완전히 독립적인 모바일 햄버거 메뉴 구동 방식처럼 변경하여 화면 활용도를 100%로 끌어올림.
- **주요 사항**: 사이드바 레이아웃 박스를 오버레이 팝업 형태로 전환하고, 작동 버튼을 헤더(Header) 메인 메뉴로 이전함.

## 변경 세부 내용
1. **버튼 위치 이전 (HTML)**
   - `index.html`: 앱 좌측 최상단 로고 바로 옆(`.header-left`)에 언제 어디서든 누를 수 있는 직관적인 `📁 탐색기` 토글 버튼(`btn-file-menu`)을 추가함.
   - 기존의 거대한 사이드바 안쪽에 지저분하게 붙어있던 접기/펼치기 토글 막대 영역(`.sidebar-header`)은 과감히 삭제하여 클린업 수행.
2. **오버레이 드로어 시각 효과 (CSS)**
   - `css/layout.css`: `.sidebar` 클래스의 구조를 DOM Flow에서 떼어내기 위해 `position: absolute`와 높은 `z-index: 1050`을 부여.
   - 닫혀있을 때 화면 좌측으로 완전히 미끄러져 나가는 `transform: translateX(-100%);` 애니메이션을 적용해, 켜졌을 땐 에디터 화면 위를 덮고(Shadow 추가) 꺼졌을 땐 화면을 1px도 낭비하지 않는 Full Screen 에디터 환경을 조성함.
3. **JS 토글 액션 및 외부 클릭 감지 추가 (JS)**
   - `js/layout.js`: 메뉴 토글 시 기존 Sidebar State를 반전시켜 열리고 닫히는 동작을 연동시킴.
   - **기능 추가**: 드로어가 열려 있을 때 화면의 빈 공간(탐색기 이외의 바깥 영역)을 클릭하면 모바일 앱처럼 팝업 형태의 탐색기가 알아서 닫히도록 `document` 전역에 `Click` 리스너를 바인딩하여 직관적인 조작감을 제공.

## 성과
- 에디터와 프리뷰 코딩 영역이 모니터를 끝까지 꽉 채우게 됨 (사이드바가 차지하던 여백 제거).
- 메뉴 클릭 시에만 탐색기가 스르륵 등장하는 트렌디한 네비게이션 구조 확립.
