# 작업 로그: 2026-04-04 패널 숨김 UI 최적화 (세로 바 안착형 기둥 디자인 적용)

## 개요
- **목적**: 창을 접었을 때 영역을 허공에 완전히 날리고 버튼만 덩그러니 남는 어색한 디자인에서, 전문 IDE 환경처럼 얇은 48px 사이즈의 세로 기둥 컨테이너로 접고 그 틀 내부에 열기 버튼을 위치시켜 시각적 안정감을 줌.
- **범위**: 에디터 구역(`editor-section`) 및 미리보기 구역(`preview-section`) 컴포넌트 확장성 적용.

## 수정/구현 작업
1. **패널 숨김 방식 개편 (`css/layout.css`)**
   - 접혔을 때의 `display: none` 옵션과 `flex: 0`을 완전히 삭제하고, `flex: 0 0 48px !important`를 부여하여 화면의 좌/우 가장자리에 무조건 48px 두께로 남도록 강제 처리. 
   - 배경 색상을 기본 `var(--bg-secondary)`로 처리해 공간이 허전하지 않고 서랍 형태의 파티션처럼 보이도록 함.
   - 단, 열었을 때 보여야 할 편집기 안의 글자, 탭, 브라우저 등의 부모 돔(Header, Container)들은 `display: none;`으로 가림.
2. **복원 버튼 착시(Embedded) 구현 (`css/panels.css`)**
   - 앞선 작업에서 스크린 중앙(`top: 50%`)으로 배치해 둔 메인 복원 버튼들을 세밀하게 오프셋(offset) 맞춤 조절.
   - 48px 패널 너비 안에, 40px 버튼 너비가 들어가야 완벽한 중앙 정렬이 되므로 `left: 4px;`와 `right: 4px;`의 CSS를 각각 하드코딩. 
   - 결과적으로 DOM 요소는 `.main-content` 전역 공간에 Floating 구조로 떠 있지만, 시각적으로는 `48px` 서랍의 가운데에 안착하고 붙어 있는 착시 구성을 활용해 속도 문제와 렌더링을 완전히 분리해줌.

## 참고 사항
- 레이아웃을 좌우가 아닌 상하(수직)로 쓸 경우 해당 창들은 높이가 48px인 띠 모양으로 변경됨. 이럴 때도 중앙 정렬이 적용됨. 완성도 높음!
