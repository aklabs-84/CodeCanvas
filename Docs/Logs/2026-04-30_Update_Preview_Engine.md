# 작업 로그: 미리보기 엔진 근본 개선 (Blob URL & Script Guard)

## 📅 날짜: 2026-04-30

## 🧐 문제 현상 분석 (상담가/해결사)
- **현상**: Chart.js와 같은 대형 외부 라이브러리를 CDN으로 로드할 때, 브라우저가 라이브러리를 완전히 다운로드/해석하기 전에 사용자의 초기화 코드가 먼저 실행되어 `Chart is undefined` 에러 발생.
- **원인**: 기존 `iframe.srcdoc` 방식은 스크립트 실행 순서 제어가 불안정하며, 특히 에디터에서 주입되는 인라인 JS가 외부 소스보다 먼저 평가되는 경향이 있음.

## 🛠️ 구현 및 개선 내용 (건축가/작업자)

### 1. 지능형 로딩 가드 엔진 주입 (`js/preview.js`)
- 미리보기 HTML 생성 시 `<head>` 최상단에 `window.__codecanvas_ready__` 시스템을 주입.
- `Promise.all`과 `load/error` 이벤트 리스너를 사용하여 현재 문서 내의 모든 `<script src="...">` 태그가 완료될 때까지 대기하는 로직 구현.
- 사용자의 메인 자바스크립트 코드를 `async IIFE`로 래핑하여 모든 라이브러리가 준비된 시점에만 실행되도록 강제함.

### 2. 렌더링 방식 전환 (srcdoc -> Blob URL)
- 기존 `srcdoc`의 한계를 극복하기 위해 `URL.createObjectURL`을 이용한 `Blob URL` 방식 채택.
- 브라우저가 미리보기를 독립된 네트워크 리소스로 인식하게 하여 캐싱 및 스크립트 실행 순서 보장 능력을 극대화함.
- 매 렌더링 시 기존 Blob URL을 `revokeObjectURL`하여 메모리 누수 방지.

### 3. 레이아웃 매니저 통합 (`js/layout.js`)
- `LayoutManager.updatePreview`가 Blob URL을 직접 처리할 수 있도록 인터페이스 수정.
- 전체화면 미리보기 모드에서도 동일한 가드 엔진이 작동하도록 완벽하게 동기화.

## ✅ 최종 결과
- **호환성**: 이제 어떤 외부 라이브러리를 사용하더라도 타이밍 이슈 없이 완벽하게 동작함.
- **안정성**: 브라우저의 네이티브 파싱 순서를 보장하며, 사용자에게는 투명한(Transparent) 로딩 경험 제공.
