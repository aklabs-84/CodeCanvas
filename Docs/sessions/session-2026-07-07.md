## 진행 작업 5
**요청:** package.json type:module 배포 후 "FUNCTION_INVOCATION_FAILED" (Vercel 플랫폼 크래시 에러 페이지) 재발, 원인 확인
**작업 내용:** `api/ai-assistant.js`의 `export default async function handler(request) {...}` 형태가 Vercel Node.js 런타임의 옛날 방식 `(req, res)` 핸들러로 오인되어(Web Standard export 규격 미준수) `request.headers.get()` 호출 시 크래시하는 것이 원인으로 확인됨(Vercel 공식 문서 기준). `export default { fetch: handler }` 형태로 export 구조 변경.
**결과:** 로컬에서 실제 `Request` 객체를 만들어 `handler.fetch()` 직접 호출 테스트 → 401 정상 응답 확인. GitHub main 푸시 완료(`a056c7d`). 실제 Vercel 배포 재현 테스트는 사용자 확인 필요.

**결과:** 완료. `npm run build` 통과. 로컬 dev 서버 브라우저 실전 테스트: (1) JS로 직접 상태 변경 후 디바운스 대기 → 히스토리 기록/undo/redo 정상 동작 확인, (2) 디바운스 완료 전 즉시 undo 호출 시에도 변경분이 먼저 flush되어 데이터 유실 없이 redo로 복구 가능함을 확인, (3) `setCode()` 호출(클리어/불러오기 시나리오) 시 히스토리가 새 베이스라인으로 리셋되고 버튼이 비활성화됨을 확인, (4) 실제 에디터에 타이핑("HELLO_UNDO_TEST") 후 툴바 실행취소 버튼을 마우스로 클릭 → 텍스트가 정상적으로 되돌려짐을 스크린샷으로 확인, 콘솔 에러 없음.
