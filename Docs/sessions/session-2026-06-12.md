## 11:00 작업 — 리사이저 왼쪽 드래그 버그 수정
**요청:** 가로 레이아웃에서 리사이저를 왼쪽으로 드래그하면 미리보기 창이 넓어지지 않는 문제 수정
**작업 내용:**
1. `css/layout.css` — `.editor-section`, `.preview-section`에 `min-width: 0; overflow: hidden;` 추가
2. `css/layout.css` — `transition: all` → `transition: flex, background-color`로 한정 (드래그 중 애니메이션 렉 방지)
3. `css/layout.css` — `body.resizing .editor-section, body.resizing .preview-section { transition: none !important; }` 추가
**원인 분석:**
- CSS flex item 기본값 `min-width: auto` → 탭+버튼 행의 자연 최소 너비(~340px) 이하로 에디터가 줄어들지 않음
- `transition: all`이 드래그 중 `flex` 값 변경을 애니메이션하여 마우스를 따라오지 못함
**결과:** 완료
- `min-width: 0px` 적용 확인 ✅
- 왼쪽 드래그: 에디터 600px → 382px, 미리보기 596px → 814px (30:70 비율) ✅
- Playwright 시뮬레이션 및 스크린샷 확인 ✅
